Старт проекта

После того как вам поступили макеты и ТЗ, нужно создать репозиторий на битбакете и добавить стартовый проект, в котором заложена структура.

В этом документе будут рассматриваться две структуры:

  1. PSD to HTML.
  2. Wordpress Themes.

Все проекты должны следовать Code Guide

Предварительные знания:

  1. Git
  2. Шпаргалка по Git
  3. Работа с консолью
  4. Работа с консолью в Windows
  5. Node.JS
  6. NPM
  7. Gulp

Последовательность действий:

Старт проекта PSD to HTML:

После того как вы в консоле перешли в папку с проектами, далее нужно клонировать стартовый проект:

параметр project-name переименуйте на свой

git clone https://github.com/redwon/start-frontend-project project-name

Переходим в папку с проектом в консоле

cd project-name

далее выполняем ряд команд

rm -rf .git

  • удалить папку гита стартового проекта (эта команда работает на линуксе или в cmder)
  • для windows использовать команду rd /s /q ".git"

git init

  • инициируем гит для нового проекта

git add -A

  • добавим стартовый проект в гит нового проекта

git commit -m "init"

  • создаем коммит инициализации

git remote add origin передаем адрес репозитория который вы создали

  • добавляем адрес удаленного репозитория
  • пример: git remote add origin https://nickname@bitbucket.org/f5-team/repo.git

git push --set-upstream origin master

  • заливаем первый коммит на битбакет и назначим по умолчанию ветку мастер

Как работать с стартовым проектом - читаем README.md стартового проекта в данном случае здесь - https://github.com/redwon/start-frontend-project

Внимание!

На сервер заливаем только файлы с папки build

Старт проекта Wordpress Themes:

После того как вы в консоле перешли в папку с проектами, далее нужно клонировать стартовый проект:

параметр domain-name переименуйте на свой домен для разработки

git clone https://github.com/redwon/wordpress-starter-theme domain-name

В созданную папку ставим Wordpress перед установкой почитать WP Secure

Переименовываем название папки темы на название проекта и в style.css темы тоже меняем название

Переходим в папку с проектом в консоле:

cd domain-name

далее выполняем ряд команд:

rm -rf .git

  • удалить папку гита стартового проекта (эта команда работает на линуксе или в cmder)
  • для windows использовать команду rd /s /q ".git"

git init

  • инициируем гит для нового проекта

git add -A

  • добавим стартовый проект в гит нового проекта

git commit -m "init"

  • создаем коммит инициализации

git remote add origin передаем адрес репозитория который вы создали

  • добавляем адрес удаленного репозитория
  • пример: git remote add origin https://nickname@bitbucket.org/f5-team/repo.git

git push --set-upstream origin master

  • заливаем первый коммит на битбакет и назначим по умолчанию ветку мастер

Как работать с стартовым проектом - читаем README.md стартового проекта в данном случае здесь - https://github.com/redwon/wordpress-starter-theme

Внимание!

Не заходите в админку через localhost только через домен.

Не заливайте папку node_modules на сервер! Исключите ее перед созданием архива.
